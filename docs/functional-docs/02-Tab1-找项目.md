# Tab1 找项目

## 功能定位

“找项目”是小程序主入口，负责把用户的个人条件转化为可执行的项目机会。该 Tab 整合用户画像问卷、AI 项目推荐、推荐结果列表和机会卡详情入口。

## 目标用户

- 不知道该做什么项目的个人独立开发者。
- 想根据技能、预算和时间筛选方向的小团队。
- 希望快速找到副业项目的探索者。

## 页面结构

### 首页状态

- 已填写画像：展示最近一次推荐结果、重新推荐入口、画像摘要。
- 未填写画像：展示问卷入口和少量示例机会卡。

### 画像问卷

问卷字段：

- 意向产品类型。
- 技能栈。
- 团队规模。
- 预算范围。
- 每周可投入时间。
- 偏好平台：App、小程序、Web、插件、SaaS、AI 工具等。
- 偏好市场：ToC、ToB、开发者工具、垂直行业、内容工具等。
- 变现偏好：订阅、单次付费、广告、电商、服务、企业版等。
- 风险偏好：稳妥、均衡、激进。

### 推荐结果页

- 每次展示 3-5 个项目机会。
- 结果以机会卡列表展示。
- 支持重新生成。
- 支持按“最适合我”“最快验证”“最低成本”“变现潜力”排序。

### 机会卡详情

详情页展示完整机会卡字段，并提供：

- 收藏。
- 加入对比。
- 生成 PRD。
- 生成 MVP 清单。
- 生成深度报告。

## 核心流程

1. 用户进入找项目。
2. 系统判断用户是否已有画像。
3. 无画像用户进入问卷。
4. 用户提交画像。
5. 后端基于画像和项目库生成候选项目。
6. AI 补全机会卡解释、MVP 范围、风险提示和推荐理由。
7. 前端展示 3-5 个推荐结果。
8. 用户选择收藏、对比或生成工作台内容。

## 推荐规则

### 输入

- 用户画像。
- 项目机会库。
- 榜单和市场雷达信号。
- 用户历史收藏和生成行为。

### 输出

- 至少 3 个项目机会。
- 每个项目必须包含推荐理由、MVP 范围、预算估算和风险提示。
- 推荐结果应尽量覆盖不同方向，避免 3 个项目过于相似。

### 适配评分

适配评分建议由以下维度组成：

- 技能匹配。
- 预算匹配。
- 时间匹配。
- 团队规模匹配。
- 变现偏好匹配。
- 市场机会匹配。
- 风险承受匹配。

## 权益与限制

### 免费用户

- 每日 3 次基础项目推荐。
- 可查看基础机会卡。
- 可收藏 10 个项目。
- 每月 1 次简单 PRD。

### Pro 用户

- 每日 20 次项目推荐。
- 可查看完整机会卡。
- 收藏不限。
- 可使用完整 PRD/MVP 生成额度。

### Deep 用户

- 高额度项目推荐。
- 可生成深度报告、冷启动方案和商业模式分析。

## 异常状态

- 问卷未完成：提示补全必填项。
- 生成失败：允许重新生成。
- 额度不足：引导购买会员或单次报告。
- 推荐结果不足：展示至少 1 个推荐，并说明“当前条件较窄，可放宽预算/平台/市场偏好”。

## 验收标准

- 用户可以在 3 分钟内完成画像问卷。
- 提交问卷后能返回至少 3 个项目机会，极端条件除外。
- 每个机会卡包含推荐理由、MVP 范围、预算估算和风险提示。
- 用户可以从推荐结果执行收藏、加入对比、生成 PRD、生成 MVP 清单。
- 重新推荐会保留用户画像，不要求重复填写全部问卷。
