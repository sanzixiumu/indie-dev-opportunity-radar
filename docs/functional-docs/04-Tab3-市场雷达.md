# Tab3 市场雷达

## 功能定位

“市场雷达”展示 App Store、安卓、小程序平台和海外平台动态，用市场信号增强项目推荐可信度。该 Tab 的重点不是做专业数据平台，而是把外部产品动态翻译成独立开发者能理解的项目机会。

## 覆盖平台

MVP 阶段建议覆盖：

- App Store。
- 国内安卓应用市场。
- 微信小程序。
- Product Hunt。
- Chrome Web Store。

v1.1 及以后可扩展：

- 抖音小程序。
- 支付宝小程序。
- Shopify App Store。
- WordPress 插件市场。
- Steam 小工具或独立游戏相关平台。
- 其他适合独立开发者观察的海外平台。

## 页面结构

### 市场雷达首页

- 平台分类入口。
- 今日/本周值得关注动态。
- 热门产品动态。
- 新上架产品观察。
- 上升榜项目追踪。

### 平台详情页

- 平台名称。
- 平台说明。
- 最新动态列表。
- 适合独立开发者的机会总结。
- 数据更新时间。

### 动态详情页

字段：

- 来源平台。
- 产品名称。
- 产品链接或来源说明。
- 观察信号。
- 版本更新摘要。
- 评论痛点摘要。
- 定价和商业模式观察。
- AI 分析。
- 适合团队类型。
- 可切入方向。
- 风险提示。
- 更新时间。

## AI 分析结构

每条市场动态的 AI 分析应包含：

- 为什么值得关注。
- 独立开发者能否切入。
- 适合什么类型团队。
- 可能的 MVP 方向。
- 潜在风险是什么。
- 与已有机会卡的关联。

## 数据策略

### MVP 阶段

- 采用人工精选 + 半自动更新。
- 不承诺实时抓取全平台数据。
- 每条动态必须标注来源和更新时间。
- 对信息不足的内容标注“信息有限”。

### v1.0 后

- 根据用户需求和数据成本接入第三方数据 API。
- 优先选择合规、稳定、可商用的数据来源。
- 对外部平台数据采集需遵守平台服务条款。

## 与其他 Tab 的关系

- 市场动态可关联机会卡。
- 用户可从市场动态生成项目机会。
- 用户可从市场动态跳转到工作台生成竞品对比报告。
- 我的 Tab 记录用户收藏过的市场动态或由动态生成的项目。

## 权益与限制

### 免费用户

- 查看市场雷达摘要。
- 查看部分平台动态。

### Pro 用户

- 查看市场雷达基础分析。
- 可收藏动态。
- 可由动态生成机会卡。

### Deep 用户

- 查看深度竞品分析。
- 生成竞品对比报告。
- 生成商业模式分析和冷启动方案。

## 风险控制

- 不将未经验证的市场判断表述为确定事实。
- 外部数据来源不稳定时，需要展示更新时间和信息完整性。
- 对平台政策、版权和数据授权保持合规。
- 对竞品评论和用户反馈做摘要时避免泄露个人隐私。

## 验收标准

- 市场雷达按平台分类展示。
- 每条动态包含来源平台、产品名称、观察信号、AI 分析和更新时间。
- 用户可以从动态进入机会卡或工作台。
- 信息不足时明确标注。
- MVP 不要求实时数据，但需要每周稳定更新精选内容。
