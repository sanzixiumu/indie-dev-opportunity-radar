# AI 与数据能力

## 功能定位

AI 与数据能力是产品的底层引擎，负责项目推荐、机会卡生成、PRD/MVP 生成、市场动态分析、对比报告和质量评估。

## AI 推荐引擎

### 输入

- 用户画像。
- 偏好条件。
- 项目机会库。
- 榜单数据。
- 市场雷达信号。
- 用户历史行为。

### 输出

- 3-5 个项目机会。
- 每个项目机会包含完整机会卡字段。
- 每个推荐结果都需要解释为什么适合该用户。

### 推荐能力

- 用户条件匹配。
- 项目可行性评估。
- 市场证据摘要。
- 风险提示。
- MVP 范围裁剪。
- 多样性控制。

## AI 内容生成

### 生成类型

- PRD。
- MVP 清单。
- 项目对比报告。
- 开发 Prompt。
- 冷启动方案。
- 商业模式分析。
- 深度项目报告。

### 通用要求

- 输出结构固定。
- 不得编造明确数据来源。
- 对信息不完整的内容标注“信息有限”。
- 必须区分事实、推断和建议。
- 生成内容需要保存到用户历史记录。

## 数据来源

### MVP 阶段

- 人工整理项目机会库。
- 人工精选榜单和平台动态。
- 半自动整理公开可访问数据。
- 用户反馈数据。

### 后续阶段

- 合规第三方数据 API。
- 公开平台搜索、榜单、RSS 或授权数据。
- 用户项目复盘数据。
- 收藏、点击、生成等行为数据。

## 项目机会库

### 字段

- 项目名称。
- 项目简介。
- 适合人群。
- 技能标签。
- 平台标签。
- 预算标签。
- 时间标签。
- 市场标签。
- 变现标签。
- 风险标签。
- 竞品参考。
- 来源说明。
- 更新时间。

### 运营规则

- 每个机会必须经过人工审核后进入正式库。
- 过期机会需要标注状态或下架。
- 高风险项目需要明确风险提示。

## 评估策略

### 推荐质量

- 推荐结果与用户画像匹配度人工评审 >= 80%。
- 用户点击机会卡详情率 >= 40%。
- 用户收藏率 >= 20%。
- 用户对推荐结果“不相关”反馈率 <= 15%。

### PRD/MVP 生成质量

- 生成内容必须包含固定结构字段。
- 生成内容不得出现明显矛盾。
- MVP 清单必须区分“必须做”和“暂不做”。
- 人工抽样评审通过率 >= 85%。

### 市场分析质量

- 每条市场分析必须包含来源、时间、观察信号和结论。
- 不允许把未经验证的市场判断表述为确定事实。
- 数据不完整时必须标注“信息有限”。

## 成本控制

- 免费用户使用较低生成额度。
- 深度内容仅在付费后生成。
- 相同输入短时间内优先复用缓存。
- 生成失败不扣减额度。
- 对长报告设置生成长度和调用成本上限。

## 风险控制

- 避免违规抓取外部平台数据。
- AI 生成内容不能替代法律、财务或投资建议。
- 对平台政策风险、版权风险、数据来源不明风险进行提示。
- 用户个人信息不应被传入不必要的第三方服务。

## 验收标准

- 推荐引擎能基于用户画像输出至少 3 个机会。
- 生成内容具有稳定结构。
- 市场分析保留来源和更新时间。
- 反馈“不相关”的推荐可以被记录。
- 额度扣减、缓存复用和失败重试逻辑生效。
