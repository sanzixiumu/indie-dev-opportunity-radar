# 独立开发者项目机会雷达 PRD

版本：v0.1  
日期：2026-06-01  
产品形态：微信小程序  
目标阶段：MVP 到 v1.0

## 1. Executive Summary

### Problem Statement

国内不少独立开发者有自主创业和开发产品的意愿，但普遍缺少高质量选题来源、市场验证线索和可执行的落地路径。现有工具多集中在应用数据、ASO、项目灵感库或开发者社区，较少把“个人条件匹配、市场证据、竞品参考、PRD/MVP 输出”整合成一个完整决策流程。

### Proposed Solution

开发一款面向独立开发者的小程序，通过用户画像问卷、AI 项目推荐、机会榜、市场雷达、PRD/MVP 工作台和收藏沉淀，帮助用户从市场信号中找到适合自己做的项目，并生成可执行的 MVP 方案。

### Success Criteria

- MVP 上线后 30 天内获取 1,000 名注册用户。
- 用户完成画像问卷后的推荐生成完成率 >= 70%。
- 推荐机会卡收藏率 >= 20%。
- PRD/MVP 生成功能使用率 >= 15%。
- 付费转化率在 MVP 阶段达到 3%-5%。

## 2. User Experience & Functionality

### User Personas

**个人独立开发者**

- 具备前端、后端、全栈、AI 应用、设计或运营能力。
- 希望找到低成本、可验证、能快速上线的项目方向。
- 痛点是想法多但难判断，不清楚市场空间、竞品情况和第一版应该做什么。

**小团队创业者**

- 2-5 人团队，有一定开发和运营能力。
- 希望从多个方向中筛选更适合团队资源的项目。
- 需要项目对比、竞品分析、PRD 和 MVP 清单辅助决策。

**副业探索者**

- 有全职工作，业余时间有限。
- 希望找到可在 7-30 天内验证的小项目。
- 更关注预算、开发周期、获客方式和变现路径。

1. 找项目

### Tab 1: 找项目

该 Tab 整合用户画像、AI 推荐和机会卡，是产品主入口。

#### Core Features

  - 意向产品类型
  - 技能栈

  - 偏好市场：ToC、ToB、开发者工具、垂直行业、内容工具等
  - 变现偏好：订阅、单次付费、广告、电商、服务、企业版等
- AI 根据用户画像推荐 3-5 个项目机会。
- 每个推荐结果展示为“机会卡”。
- 机会卡支持收藏、加入对比、生成 PRD、生成 MVP 清单。

### User Stories

**Story 1: 项目推荐**

As a 独立开发者, I want to 填写自己的技能、预算和时间, so that 我可以获得适合自己条件的项目方向。

Acceptance Criteria:

- 用户可以在 3 分钟内完成画像问卷。
- 系统每次至少返回 3 个项目机会。
- 每个项目机会必须包含推荐理由、MVP 范围、预算估算和风险提示。
- 用户可以对推荐结果执行收藏、对比和生成操作。

**Story 2: 机会榜浏览**

As a 副业探索者, I want to 每周查看适合独立开发者的新项目榜单, so that 我可以持续获得选题灵感。

Acceptance Criteria:

- 机会榜至少每周更新 1 次。
- 每个榜单项目必须有详情页。
- 详情页可以跳转到机会卡，并支持收藏和加入对比。

**Story 3: 市场雷达**

As a 小团队创业者, I want to 查看不同平台上的产品动态和竞品信号, so that 我可以判断项目是否有真实市场需求。

Acceptance Criteria:

- 市场雷达按平台分类展示。
- 每条市场动态必须包含来源平台、产品名称、观察信号和 AI 分析。
- 系统需要标注信息更新时间。

**Story 4: 工作台落地**

As a 准备启动项目的开发者, I want to 把感兴趣的方向生成 PRD、MVP 清单和对比报告, so that 我可以快速进入开发和验证。

Acceptance Criteria:

- 用户可以从机会卡一键生成 PRD。
- 用户可以从机会卡一键生成 MVP 清单。
- 用户可以选择 2-3 个机会卡进行对比。
- 生成内容支持保存到“我的”。

**Story 5: 收藏与历史**

As a 回访用户, I want to 找回之前收藏和生成过的项目内容, so that 我可以继续完善决策。

Acceptance Criteria:

- 用户可以在“我的”查看收藏项目。
- 用户可以查看历史 PRD 和 MVP 清单。
- 用户可以修改个人画像并重新生成推荐。

### Non-Goals

MVP 阶段不做以下内容：

- 不做全平台实时数据抓取。
- 不做完整 ASO 专业分析工具。
- 不做开发者社区和即时聊天。
- 不做项目众筹、融资或合伙人撮合。
- 不做自动生成完整可上线代码。
- 不做企业级 BI 数据看板。

## 3. AI System Requirements

### Tool Requirements

#### AI 推荐引擎

- 输入：用户画像、偏好条件、平台趋势、项目库标签。
- 输出：3-5 个机会卡。
- 关键能力：
  - 用户条件匹配
  - 项目可行性评估
  - 市场证据摘要
  - 风险提示
  - MVP 范围裁剪

#### AI 内容生成

- PRD 生成
- MVP 清单生成
- 项目对比报告生成
- 开发 Prompt 生成
- 冷启动方案生成

#### 数据来源

MVP 阶段：

- 人工整理项目机会库。
- 人工精选榜单和平台动态。
- 半自动整理公开可访问数据。

后续阶段：

- 接入合规第三方数据 API。
- 接入公开平台搜索、榜单、RSS 或授权数据。
- 建立用户反馈数据闭环。

### Evaluation Strategy

#### 推荐质量评估

- 推荐结果与用户画像匹配度人工评审 >= 80%。
- 用户点击机会卡详情率 >= 40%。
- 用户收藏率 >= 20%。
- 用户对推荐结果“不相关”反馈率 <= 15%。

#### PRD/MVP 生成质量评估

- 生成内容必须包含固定结构字段。
- 生成内容不得出现明显矛盾。
- MVP 清单必须区分“必须做”和“暂不做”。
- 人工抽样评审通过率 >= 85%。

#### 市场分析质量评估

- 每条市场分析必须包含来源、时间、观察信号和结论。
- 不允许把未经验证的市场判断表述为确定事实。
- 对数据不完整的情况必须标注“信息有限”。

## 4. Technical Specifications

### Architecture Overview

#### Client

- 微信小程序端。
- 五个底部 Tab：找项目、机会榜、市场雷达、工作台、我的。
- 支持微信登录、收藏、历史记录、会员状态展示。

#### Backend

- 用户画像服务。
- 项目机会库服务。
- 推荐服务。
- AI 生成服务。
- 榜单和市场动态服务。
- 收藏与历史记录服务。
- 支付与会员服务。

#### Data Flow

1. 用户进入“找项目”并填写画像。
2. 前端提交画像到后端。
3. 后端读取项目机会库和标签体系。
4. 推荐服务生成候选项目。
5. AI 生成服务补全机会卡解释、MVP 范围和风险提示。
6. 前端展示 3-5 个机会卡。
7. 用户收藏、对比或生成 PRD/MVP。
8. 生成结果保存到用户资产。

### Integration Points

- 微信小程序登录。
- 微信支付。
- AI 模型 API。
- 内容管理后台。
- 第三方应用数据 API，v1.0 后按需接入。
- 对象存储或数据库，用于保存生成记录和导出文件。

### Security & Privacy

- 用户画像、收藏和生成记录默认仅用户本人可见。
- 支付信息通过微信支付处理，产品侧不保存银行卡等敏感支付数据。
- AI 请求中避免传递不必要的个人身份信息。
- 市场雷达数据必须保留来源和更新时间。
- 对外部平台数据采集需遵守平台服务条款，避免违规爬取。

## 6. Growth & Operations

### Positioning

帮独立开发者从市场信号里找到适合自己做的项目，并生成可执行的 MVP 方案。

### Content Strategy

#### Column 1: 每周项目机会榜

示例标题：

- 本周 10 个适合一人开发的小项目
- 7 天内可以验证的 AI 工具方向
- 适合小团队切入的小程序项目

#### Column 2: 竞品拆解

示例标题：

- 一个增长很快的记账 App，独立开发者还能不能做？
- 从某个热门小程序看轻量工具的机会
- Product Hunt 上升产品背后的独立开发机会

#### Column 3: 从选题到 MVP

示例标题：

- 用 7 天验证一个 Chrome 插件项目
- 一个 AI 小工具从选题到上线的完整拆解
- 如何把一个榜单趋势变成可开发产品

### Channel Strategy

- 即刻：触达独立开发者、AI 产品人、早期创业者。
- V2EX：触达技术用户，内容需要真诚、具体、少营销。
- 掘金：发布开发者教程和项目拆解。
- 小红书：触达副业、AI 工具和轻创业人群。
- 公众号：沉淀长文、榜单和深度案例。
- 视频号/B 站：发布项目拆解、选题复盘和案例讲解。
- 微信群：获取种子用户和高质量反馈。

### Cold Start Flow

1. 每周发布一期《独立开发机会榜》。
2. 内容中植入“测测你适合做什么项目”的小程序入口。
3. 用户进入小程序完成画像问卷。
4. 系统生成 3-5 个机会卡。
5. 引导用户收藏、对比或生成 PRD。
6. 将深度报告、完整 PRD、竞品对比作为付费转化点。

## 7. Risks & Roadmap

### Phased Rollout

#### MVP

- 用户画像问卷。
- AI 推荐 3 个项目机会。
- 机会卡详情。
- 机会榜。
- 收藏。
- 简单 PRD/MVP 清单生成。
- 基础会员和单次付费。
- 市场雷达采用人工精选和半自动更新。

#### v1.1

- 完整项目对比。
- 更丰富的机会榜分类。
- PRD/MVP 导出。
- 用户反馈闭环。
- 市场雷达增加固定更新节奏。
- 开发 Prompt 生成。

#### v2.0

- 接入合规第三方数据 API。
- 建立项目适配评分模型。
- 建立小团队可做指数。
- 建立冷启动难度指数。
- 增加项目持续跟踪。
- 增加更细分行业机会库。

### Technical Risks

**数据合规风险**

- 外部平台数据采集可能受到服务条款限制。
- 应对方式：MVP 阶段以人工精选和合规公开数据为主，商业化后优先考虑授权数据或第三方 API。

**AI 推荐质量不稳定**

- 推荐可能过泛、不可执行或与用户条件不匹配。
- 应对方式：建立结构化机会库、固定机会卡字段、人工抽样评估和用户反馈机制。

**生成成本风险**

- PRD、MVP 和竞品分析生成可能带来较高模型调用成本。
- 应对方式：设置额度、缓存结果、区分免费和付费模型调用深度。

**冷启动内容成本**

- 机会榜和市场雷达前期需要人工运营。
- 应对方式：先聚焦 3-5 个高价值栏目，不追求全平台覆盖。

**用户付费意愿不确定**

- 用户可能愿意看免费项目灵感，但不一定愿意付费。
- 应对方式：优先验证 9.9 元单次深度报告和 199 元年费会员。

## 8. Open Questions

- 产品正式名称是否使用“独立开发者项目机会雷达”，还是改为更短的小程序名称。
- 第一版是否只做微信小程序，还是同步做 Web 落地页用于 SEO 和内容承接。
- 市场雷达首批重点平台是 App Store、小程序、Product Hunt，还是优先聚焦其中 1-2 个。
- AI 模型供应商、预算上限和生成频控策略待确认。
- 是否需要在 MVP 阶段提供人工选题诊断服务。
