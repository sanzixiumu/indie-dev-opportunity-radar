# Tab1 AI 灵感孵化设计

日期：2026-06-01

## 背景

Tab1 原功能文档将“找项目”设计为画像问卷 + AI 推荐。新的产品方向改为“灵感孵化”：用户先输入一个项目想法，大模型按想法动态追问关键信息，信息足够后进行市场调研和方向分析，最终把确定方向保存到工作台。

本设计只定义 Tab1 前端功能、UI 摆放、AI 弹窗流程、结果结构和工作台衔接。实际大模型接口、联网调研能力和提示词服务可在后续实现计划中拆分。

## 目标

- 让用户不用填写传统问卷，也能通过自然输入启动项目方向分析。
- 用全屏 AI 工作弹窗承载追问、调研进度和分析结果，减少页面跳转和突兀弹窗感。
- 让大模型动态生成追问问题和结果内容，前端只负责稳定渲染结构。
- 将用户确定的方向保存为工作台中的“已生成项目”，由工作台承接收藏、对比、生成 PRD 和详情查看。

## 非目标

- Tab1 不承担收藏、对比、生成 PRD 等深度项目操作。
- Tab1 不展示完整机会榜或市场雷达内容。
- 本阶段不在前端硬编码固定的 4-6 个追问问题。
- 本设计不实现真实联网调研，只定义前端状态和结构化输出契约。

## Tab1 初始页

Tab1 初始页保持轻量，只服务一个核心动作：输入项目想法并开始孵化。

页面从上到下：

1. 顶部标题区
   - 标题：`灵感孵化`
   - 副标题：`输入一个项目想法，AI 会追问关键信息，并调研国内外产品，帮你找到可切入方向。`
2. 灵感输入区
   - 多行输入框。
   - placeholder：`例如：我想做一个帮独立开发者找项目方向的小工具`
   - 主按钮：`开始孵化`
   - 未输入时按钮置灰，输入后可点击。
3. 示例灵感区
   - 展示 3-4 个可点击示例，点击后填入输入框。
   - 示例可包括：`AI 帮我整理小红书选题`、`给自由职业者做报价管理工具`、`帮程序员快速生成项目 PRD`、`面向本地商家的会员小程序`。
4. 最近孵化区
   - 有历史结果时展示最近 2-3 个已生成方向摘要。
   - 无历史结果时展示轻量空状态：`孵化完成的项目会保存到工作台。`
5. 底部提示
   - 小字提示：`市场调研结果由 AI 生成，请结合真实数据进一步验证。`

## 全屏 AI 弹窗

用户点击 `开始孵化` 后，打开一个从底部上滑进入的全屏弹窗。弹窗不是居中提示框，而是临时的 AI 工作空间。

顶部固定区域：

- 左侧关闭按钮。
- 中间标题：`AI灵感孵化`。
- 标题下方显示当前阶段，例如 `正在补充关键信息`、`正在调研市场`、`已生成方向建议`。
- 阶段提示旁展示 AI 灵感图标。
- 追问阶段可展示轻量进度，但不写死总题数，因为问题数量由大模型动态决定。

弹窗包含三种内部状态：追问态、生成进度态、分析结果态。

## 追问态

前端不固定追问题目。大模型根据用户输入的项目想法，动态生成必要问题。问题数量通常为 4-6 个，但应由大模型判断是否信息足够。

大模型追问应覆盖但不机械限定于这些信息类型：

- 目标用户与核心痛点。
- 产品平台或使用场景。
- 用户资源、能力和限制。
- 用户最看重的结果。
- 变现偏好。
- 不可接受的风险。

每个问题由大模型返回以下结构：

- `questionId`
- `title`
- `description`
- `type`：`single` 或 `multiple`
- `options`
- `allowCustomInput`
- `isRequired`

前端每轮展示一个问题卡片：

- 问题标题。
- 问题说明。
- 选项组。
- 补充输入框。
- 底部按钮：`上一步`、`下一步`。

当大模型判断信息足够时，底部主按钮变为：`开始调研市场`。

## 生成进度态

用户点击 `开始调研市场` 后，同一个全屏弹窗切换为生成进度态。

三步进度：

1. 正在联网查询国内外优秀产品
2. 正在对比产品优劣
3. 正在分析产品方向

每一步初始为 loading。完成一项后打勾，再进入下一项。全部完成后，弹窗原位切换为分析结果态。

这些进度文案代表大模型和后端正在执行的任务。若后端暂时不能返回真实阶段事件，前端可以用模拟阶段进度展示，但最终结果必须来自大模型生成。

## 分析结果态

结果内容由大模型生成，前端按固定结构渲染。结果卡片结构：

1. 方向结论
   - 说明该方向是否建议做、是否需要收窄、是否暂不建议直接做。
2. 国内产品表格
   - 字段：产品名称、产品定位、主要优势、主要短板。
   - 默认展示国内前 3 个头部或代表性产品。
3. 国外产品表格
   - 字段：产品名称、产品定位、主要优势、主要短板。
   - 默认展示国外前 3 个头部或代表性产品。
4. 推荐切入方向和可切入优势
   - 结合用户输入、追问答案和竞品差异，说明应该从哪里入手。
5. 风险
   - 每条风险必须包含风险描述和风险来源。
   - 风险来源可以是竞品强度、平台政策、获客难度、数据来源不稳定、合规要求、用户付费意愿等。
6. 建议
   - 给出下一步最小验证动作和优先级。

结果态底部固定三个按钮：

- `重新生成`：保留灵感和追问答案，重新执行市场调研和分析。
- `修改方向`：回到追问态，允许修改灵感或补充答案。
- `确定方向`：保存当前结果，并跳转到工作台项目列表。

## 工作台衔接

用户点击 `确定方向` 后，系统把当前分析保存为一个已生成项目，然后跳转到 Tab4 工作台。

工作台项目列表展示：

- 项目方向标题。
- 方向结论摘要。
- 推荐切入方向摘要。
- 生成时间。
- 主要风险数量或风险标签。
- 操作按钮：`收藏`、`对比`、`生成PRD`。

点击项目列表中的某条项目，打开详情窗口。详情窗口展示完整分析结果：

1. 方向结论。
2. 国内产品表格。
3. 国外产品表格。
4. 推荐切入方向和可切入优势。
5. 风险。
6. 建议。

## 数据结构

前端可使用统一的 `GeneratedProject` 数据对象承接 Tab1 结果和工作台列表。

```ts
type GeneratedProject = {
  id: string;
  sourceIdea: string;
  answers: IncubationAnswer[];
  title: string;
  conclusion: string;
  domesticProducts: ProductResearchItem[];
  globalProducts: ProductResearchItem[];
  entryDirection: string;
  advantages: string[];
  risks: RiskItem[];
  suggestions: string[];
  createdAt: string;
  favoriteStatus: boolean;
  compareStatus: boolean;
};

type IncubationAnswer = {
  questionId: string;
  questionTitle: string;
  selectedOptions: string[];
  customInput?: string;
};

type ProductResearchItem = {
  name: string;
  positioning: string;
  strengths: string;
  weaknesses: string;
};

type RiskItem = {
  description: string;
  source: string;
};
```

## 大模型提示词边界

前端不直接写死追问内容和结果文案，但需要和后端约定两类结构化输出。

追问提示词目标：

- 根据用户输入的项目想法，判断缺少哪些关键信息。
- 动态生成 4-6 个追问问题。
- 每个问题要适合移动端选择，不应过长。
- 问题应尽量帮助后续市场调研、竞品分析和切入建议。
- 当信息足够时返回可以开始调研的状态。

结果提示词目标：

- 基于用户想法和追问答案，调研国内外各 3 个头部或代表性产品。
- 分析每个产品的定位、优势和短板。
- 给出方向结论、推荐切入方向、可切入优势、风险和建议。
- 风险必须说明来源，不得把未经验证的推断写成确定事实。
- 输出必须匹配前端固定结构。

## 状态与异常

- 未输入项目想法：`开始孵化` 按钮不可点击。
- 用户关闭弹窗：保留当前灵感输入，追问进度可暂不保存。
- 追问生成失败：提示重试，保留用户已输入灵感。
- 市场调研生成失败：允许重新生成，不保存失败结果。
- 结果信息不足：结果中应明确标注“信息有限”，并给出需要补充的方向。
- 用户点击 `确定方向` 保存失败：提示重试，不跳转工作台。

## 验收标准

- 用户可以在 Tab1 输入项目想法并打开全屏 AI 灵感孵化弹窗。
- 弹窗从底部上滑进入，顶部显示 `AI灵感孵化` 和当前阶段。
- 追问问题由大模型动态生成，前端不硬编码固定题目。
- 信息足够时主按钮显示为 `开始调研市场`。
- 生成进度态展示三步 loading/check 状态。
- 分析结果按固定结构展示：方向结论、国内产品表格、国外产品表格、推荐切入方向和可切入优势、风险、建议。
- 风险内容展示风险来源。
- 点击 `确定方向` 后，结果保存为工作台项目并跳转到工作台列表。
